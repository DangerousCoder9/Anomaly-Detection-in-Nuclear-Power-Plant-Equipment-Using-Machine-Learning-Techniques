
## 📈 Plot 1: Training Loss Curve

**Verdict: ✅ Excellent training — well converged, no overfitting**

## 📊 Plot 2: Error Distribution Histogram

**What it shows:** The spread of reconstruction errors for training data (blue) vs all test data (orange), with the threshold marked as a red dashed line.

**Reading it:**

| Group | Error Range | Position |
|-------|------------|----------|
| Training errors (blue) | ~0.00005–0.0006 | Tight cluster LEFT of threshold |
| Test normal errors (orange, left peak) | ~0.00005–0.0007 | LEFT of threshold |
| Test anomaly errors (orange, right hump) | ~0.012–0.018 | FAR RIGHT of threshold |

**Key observations:**
- The **blue (training) bar is almost invisible** because it's completely overlapping with the normal test errors — meaning the model behaves identically on training and unseen normal data. Perfect generalization.
- There is a **clear gap** between the two orange humps (~0.001 to ~0.012) — normal and anomaly errors are **well separated** with no overlap
- The threshold (0.0007) sits **right in the middle of that gap** — it's perfectly placed to separate the two groups

**Verdict: ✅ Perfect separation between normal and anomaly — threshold is ideally positioned**

## ⏱️ Plot 3: Test Errors Over Time

**What it shows:** Reconstruction error for every test window in sequence, with true anomaly locations marked as red dots.

**Reading it:**

| Window Index | Behavior | Meaning |

| 0 → ~5800 | Flat blue line hugging zero, below threshold | All normal data, model reconstructs perfectly |
| ~4500–5800 | Small spikes (~0.0025) but still below threshold | Minor sensor fluctuations, correctly not flagged |

| 6000 → 10000 | Errors oscillate between 0.012–0.018, all above threshold | Sustained anomaly period, all correctly flagged |

**What's remarkable:**
- The model detects the attack **at almost the exact moment it starts** (~window 5800–6000)
- During the entire normal period (0–5800), errors stay flat with only tiny blips — **zero false alarms** in that region
- The red dots (true anomalies) align perfectly with where the error crosses the threshold

**Verdict: ✅ Near-perfect temporal detection — model catches the attack right as it begins**

## 🔢 Plot 4: Confusion Matrix

**What it shows:** The breakdown of every prediction the model made vs the true label.

```
                 Predicted Normal    Predicted Anomaly
True Normal           5921  ✅TN          77  ❌FP
True Anomaly             3  ❌FN        3997  ✅TP
```

**Calculating the metrics:**

| Metric | Formula | Result |
|--------|---------|--------|
| **Accuracy** | (5921+3997) / (5921+77+3+3997) | **99.2%** |
| **Precision** | 3997 / (3997+77) | **98.1%** — of all anomaly predictions, 98.1% were real |
| **Recall** | 3997 / (3997+3) | **99.9%** — of all real anomalies, 99.9% were caught |
| **F1-Score** | 2×(0.981×0.999)/(0.981+0.999) | **99.0%** |

**Breaking down the errors:**
- **77 False Positives** — 77 normal windows were incorrectly flagged as anomaly. Looking at Plot 3, these are likely the small spikes around window 4500–5800 where sensor fluctuations briefly touched near the threshold
- **3 False Negatives** — only 3 real anomaly windows were missed. These are likely the very first windows where the attack just started entering the sliding window (not enough anomalous rows yet to push the error over threshold)

**Verdict: ✅ Outstanding performance — 99.2% accuracy with near-perfect recall means almost no attacks are missed**
