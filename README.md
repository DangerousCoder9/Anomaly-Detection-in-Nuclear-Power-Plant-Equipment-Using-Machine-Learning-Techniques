# Anomaly Detection in Nuclear Power Plant Equipment Using Machine  Learning Techniques
Anomaly Detection in Nuclear Power Plant Equipment Using Machine Learning  This project focuses on detecting abnormal behavior in nuclear power plant equipment using machine learning techniques. Nuclear power plants rely on multiple sensors to continuously monitor critical parameters such as temperature, pressure, coolant flow, turbine speed, and radiation levels. Identifying unusual patterns in this sensor data is essential to ensure safety, reliability, and efficient operation.  The goal of this project is to build an intelligent anomaly detection system that learns normal operational patterns from historical sensor data and detects deviations that may indicate equipment malfunction or unsafe conditions.  ## Project Overview  The system analyzes time-series data collected from different sensors in the nuclear power plant. Each sensor records measurements at regular time intervals. These readings are processed and used to train a deep learning model capable of identifying abnormal behavior.  Instead of relying on predefined thresholds, the model learns the normal behavior of the system and detects anomalies based on reconstruction error.  ## Key Features  * Detection of abnormal patterns in nuclear plant sensor data * Time-series analysis using deep learning techniques * Automatic learning of normal equipment behavior * Early detection of potential equipment faults * Visualization of anomaly detection results  ## Dataset Structure  The dataset contains time-stamped sensor readings from different components of the nuclear power plant.  Example format:  | Timestamp | Sensor1     | Sensor2  | Sensor3   | Sensor4       | Sensor5   | Label          | | --------- | ----------- | -------- | --------- | ------------- | --------- | -------------- | | Time      | Temperature | Pressure | Flow Rate | Turbine Speed | Radiation | Normal/Anomaly |  Each row represents a sensor reading at a specific moment in time.  ## Data Preprocessing  Several preprocessing steps are performed before training the model:  * Removing the timestamp column * Normalizing sensor values to a common scale * Converting sequential data into time windows using a sliding window technique * Separating training and testing datasets  The sliding window approach allows the model to analyze temporal patterns in the sensor data.  ## Model Architecture  The anomaly detection model is based on a **Conv1D-LSTM Autoencoder architecture**, which combines convolutional layers and recurrent neural networks.  Model components include:  * **Conv1D Layer** – detects short-term temporal patterns in sensor data * **LSTM Encoder** – compresses the time-series sequence into a compact representation * **Repeat Vector** – prepares the encoded representation for reconstruction * **LSTM Decoder** – reconstructs the original sequence * **Dense Layer** – outputs reconstructed sensor values  The model learns to reconstruct normal sequences. When abnormal sequences are input, the reconstruction error increases, indicating a potential anomaly.  ## Anomaly Detection Method  The system detects anomalies using reconstruction error.  1. The model reconstructs the input sequence. 2. The reconstruction error is calculated. 3. A threshold is defined based on the mean and standard deviation of training errors.  If the reconstruction error exceeds the threshold, the sequence is classified as an anomaly.  ## Evaluation Metrics  The model performance is evaluated using:  * Precision * Recall * F1-score * Confusion Matrix  These metrics help measure how effectively the system detects anomalies while minimizing false alarms.  ## Applications  This project can be applied to:  * Nuclear power plant monitoring * Industrial equipment monitoring * Predictive maintenance systems * Fault detection in complex machinery * Safety monitoring systems  ## Technologies Used  * Python * TensorFlow / Keras * NumPy * Pandas * Matplotlib * Scikit-learn
